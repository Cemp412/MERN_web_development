Qs. What is a database?
ans. It is a collection of data in a format that can be easily accessed.

Qs1. Why databases?
ans. 1. Can store large data.
     2. features like security, scalability etc.
     3. Easier to insert, update or delete data.

Qs2. SQL vs NoSQL?
ans.    SQL                                     NoSQL
    1. Relational database                     1. Non Relational database.
    (data stored in Tables.)                (data stored in document/key-val/graphs etc.)
    eg- MySQL, Oracle, PostgresSQL etc.     eg- MongoDB, cassandra, Neo4j etc.

Qs3. What is SQL?
ans. SQL stands for (Structured Query Language)
    SQL is a programming language used to interact with Relational databases.

Qs4. What is table in SQL?
ans. In SQL, a table is a database object used to store structured data in rows and columns, similar to a spreadsheet. Tables are a fundamental component of relational databases, allowing users to organize and manage data effectively. Each table consists of:

Columns (Fields): Define the structure of the table. Each column has a name, data type, and sometimes constraints (e.g., NOT NULL, UNIQUE).

Example: id, name, age.
Rows (Records): Contain the actual data in the table. Each row represents a single record in the table.

9. Table Queries: Create, Insert, Update, Alter, Truncate, Delete

        # Create Table command: Create Table table_name (column_name1 datatype constraint, column_name2 datatype constraint);
                ex: Create Table User (id INT NOT NULL, age INT NOT NULL, name VARCHAR(30) NOT NULL, email VARCHAR(50) UNIQUE, followers INT DEFAULT 0, following INT DEFAULT 0);

        # INSERT INTO Table: INSERT INTO Table_name (column name1, column name 2) VALUES(Col1_v1, col2_v1), (col1_v2, col_2_v2);
                ex: INSERT INTO USER (id, age, name, email, followers, following) VALUES(1, "16", "Adam", "adam@email.com", 5, 8), (2, "18", "Adnan", "adnan@email.com", 23, 83);

        # UPDATE into table:  Update(to update existing rows)
                Syntax: UPDATE table_name SET col1 = val1, col2 = val2 WHERE condition;
                ex: UPDATE user SET followers = 600 WHHERE age = 16;
        ** SET SQL_SAFE_UPDATES = 0; **

        # DELETE from table: Delete(to delte existing rows).
                Syntax: DELETE from table_name where condition;
                ex: DELETE from user where age = 16;
                
        # ALTER table: It is used to change the schema. 
                1. ADD Column: ALTER Table table_name ADD COLUMN column_name datatype constraint;
                ex: ALTER Table user ADD COLUMN city VARCHAR(50) DEFAULT "Delhi";

                2. DROP Column: ALTER Table table_name DROP COLUMN column_name;
                ex: ALTER table user DROP COLUMN city; 

                3. RENAME Table: ALTER Table table_name RENAME TO new_table_name;
                ex: ALTER Table user RENAME TO instaUser;

                4. CHANGE Column(rename): ALTER Table table_name CHANGE COLUMN old_name new_name new_datatype new_constraint;
                ex: ALTER Table instaUser CHANGE COLUMN followers subscribers INT DEFAULT 0;

                5. MODIFY Column(modify datatype/constraint): ALTER Table table_name MODIFY col_name new_datatype new_constraint;
                ex: ALTER Table instaUser MODIFY subscribers INT DEFAULT 5;

                6. TRUNCATE : (to delete table's data).
                Syntax: TRUNCATE TABLE table_name;
                ex: TRUNCATE TABLE instaUser;

11. CONSTRAINTS : Rules for data in the table.
        NOT NULL : Columns cannot have a null value.
        UNIQUE : All values in column are different.
        DEFAULT: Stes the default value of a column.
        CHECK : It can limit the values allowed in a column.

        EX: i.  Salary INT DEFAULT 25000;
            ii. CONSTRAINT age_check CHECK(age >= 18 AND city = "Delhi");

        ex: Create Table User (id INT NOT NULL, age INT ,name VARCHAR(30) NOT NULL, email VARCHAR(50) UNIQUE, followers INT DEFAULT 0, following INT DEFAULT 0,  CONSTRAINT age_check CHECK(age >= 13));

12. Other Important constraint Key : 
        1. PRIMARY KEY : makes a column unique & not null but used only for one. A table can have only one primary key.
        ex: CREATE TABLE temp( id int not null, PRIMARY KEY (id));

        2. FOREIGN KEY : prevent actions that would destroy link b/w tables.
        ex: CREATE TABLE temp( cust_id int, FOREIGN KEY (cust_id) references customer(id));

13. Q. What are keys?
Ans. Keys are special column in the table.

        1. PRIMARY KEY: It is a column (or set of columns) in a table that uniquely identifies each row. There is only 1 PK & it should not be null.

        2. FOREIGN KEY (FKs) : It is a column (or a set of columns) in a table that refers to the primary key in another tables. FOREIGN Keys can have duplicate & null values. They can have multiple FKs.

14. SELECT Command: Select & show data from the DB.
        Syntax: SELECT col1, col2 FROM table_name;
        ex: SELECT id, name, age from user;
        Syntax to show all: SELECT * FROM table_name;
        ex: SELECT * from user;

15. DISTINCT: It is used to show unique records from table.
        Syntax: SELECT DISTINCT age from user;



/****************** SQL Part 2 Begins ***************************/

1. where clause: To define some conditions.
        Syntax: SELECT col1, col2 FROM table_name WHERE conditions;
        ex: SELECT * FROM user Where followers >= 200;

2. Operators in where: 
        1. Arithmetic Operators: +(addition), *(multiplication), /(division), %(modulus)
        2. Comparison Operators: =(equal to), !=(not equal to), >, >=, <=
        3. Logical Operators: AND, OR, NOT, IN, BETWEEN, ALL, LIKE, ANY
        4. Bitwise Operators: & (Bitwise AND) | (Bitwise OR)

3. Frequently Used Operators: 
        1. AND: It is used  to check for both the conditions to be true.
        2. OR: to check one of the conditions to be true.
        3. BETWEEN: selects for a given range.
        4. IN: matches any value in the list.
        5. NOT: to negate the given condition.

        ex:  1. SELECT name, age from user where age > 17 AND followers >= 20;
        2. SELECT name, age from user where age > 17 OR followers >= 20;
        3. SELECT name, age from user where age BETWEEN 17 AND 20;
        4. SELECT name, age, followers from user where email IN ("adam@email.com", "anu@email.com");
        5. SELECT name, age, followers from user where email NOT IN ("adam@email.com", "anu@email.com");

4. Limit clause : Sets an upper limit on number of tuple (rows) to be returned.
        Syntax: SELECT col1, col2 from table_name LIMIT number;
        ex: SELECT name, age from user LIMIT 2;

5. Order By clause: To sort in ascending (ASC) or descending order (DESC).
        Syntax: SELECT col1, col2 From table_name ORDER BY col_name(s) ASC;
        ex:  SELECT name, age, followers from user ORDER BY followers DESC;

6. Aggregate Functions: Aggregate functions perform a calculation on a set of values, and return a single value.
        1. COUNT() :
        2. MAX() : 
        3. MIN():
        4. SUM() :
        5. AVG() :

        ex: 1. SELECT max(marks) from student;
        2. SELECT COUNT(id) from user;
        3. SELECT min(marks) from student;
        4. SELECT sum(followers) from user;

7. GROUP BY clause : Group rows tha that have the same values into summary rows. It collect data from multiple records and groups the result by one or more column.
        Syntax: SELECT col1, col2 From table_name GROUP BY col_name(s);
        ex: SELECT age, count(id) FROM user GROUP BY age;
        SELECT age, max(followers) FROM user GROUP BY age;

** Generally we use GROUP BY with some  aggregation function. **

8. HAVING clause: Similar to where i.e. applies some condition on rows.But it is used when we want to apply any condition after grouping. 
        Syntax: SELECT col1, col2 FROM table_name GROUP BY col_name(s) HAVING condition;
        ex: SELECT age, max(followers) FROM user GROUP BY age HAVING max(followers) > 200;

** WHERE is for the table, HAVING is for group.**
** Grouping is necessary for HAVING.**

9. General Order: 
        Syntax: SELECT column(s) FROM table_name WHERE condition GROUP BY column(s) HAVING condition ORDER BY column(s) ASC;
